Chapter 2: Briefly define

  1. Natural Language: the language we speak and write in our everyday lives.
  2. Pseudocode: Plain language description fo the steps in an algorithm or another system.
  3. sequential operation (straight line algorithm): the sequential execution of operations one after another.
  4. input: Get a value
  5. output: Print a value
  6. conditional statement: They allow an algorithm to ask a yes/no question and select the next operation to perform on the basis of the answer to that question.
  7. boolean statement: an expression that evaluates to either true or false .
  8. algorithmic discovery process: Finding the solution to a given problem

Briefly describe or define:

  1. Why algorithms should be correct: Algorithms that give wrong results lead to mistakes that are expensive and/or fatal.
  2. Why algorithms should be efficient: As computer memory capacity and processing speed increase, people find more complex problems to be solved, so the boundaries of the computer's resources continue to be pushed.
  3. program maintenance: Modifying a program, either to correct errors or to expand its functionality.
  4. Importance of algorithm: ease of understanding: The program being maintained should be easy to understand.
  5. algorithmic elegance: is equivalent to style. A program with elegance can be more difficult to undertand
  6. sequential search algorithm: Searching a list of items to locate a particular item.
  7. order of magnitude: Anything that varies as a constant times n
  8. selection sort: selects the smallest (or largest) element from the unsorted portion of the list and moving it to the sorted portion of the list.
  9. flop: compares performance among different makes of computers by testing how many arithmetic operations one can do in 1 second.
  10. teraflop: 1 trillion flops
  11. petaflop: 1 quadrillion flops
  12. parallel processor: a method in computing of running two or more processors to handle separate parts of an overall task.
  13. data cleanup problem : removes incorrect, corrupted, incorrectly formatted, duplicate, or incomplete data.
  14. binary search
  15. pattern matching
